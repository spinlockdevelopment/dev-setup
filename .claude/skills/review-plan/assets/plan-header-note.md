> **Checkpoint Reviews:** This plan uses batched cross-model reviews at explicit `### Checkpoint` blocks instead of a review after every task. When executing with `superpowers:executing-plans`, run the checkpoint block when you reach it — that is the review for the preceding batch. When executing with `superpowers:subagent-driven-development`, keep the per-task spec-compliance review, but replace the per-task code-quality review with the checkpoint block (code quality is reviewed in batches instead of per-task). Per-task tests still run normally.
