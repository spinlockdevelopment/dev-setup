> **Parallel Tracks:** This plan groups tasks into independent tracks that can progress simultaneously. Execute each track in its own git worktree off the plan branch — see `superpowers:using-git-worktrees` for setup and `superpowers:dispatching-parallel-agents` for coordinating the per-track implementers. Each track has its own checkpoints (`Checkpoint A.1`, `B.1`, etc.). Checkpoint `CHECKPOINT_BASE`/`CHECKPOINT_HEAD` refer to the **track's** branch, not the plan branch — that way `/codex:review --scope branch` sees only that track's commits, never interleaved work from the other tracks. Tracks merge back to the plan branch when they are done; the final integration checkpoint at the end of the plan reviews the combined result.
